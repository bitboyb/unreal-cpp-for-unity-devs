# Contributing

Thanks for helping improve this guide.

## Good Contributions

- fix technical mistakes or outdated Unreal guidance
- simplify explanations for Unity developers who are new to C++
- add focused Unity C# and Unreal C++ examples
- improve navigation, glossary entries, or cross-links
- expand workflow notes for solo developers and teams

## Style Expectations

- keep explanations short and practical
- define Unreal terms before using them heavily
- prefer small examples over large code dumps
- use Unreal Engine 5 terminology
- avoid presenting Unity and Unreal concepts as exact one-to-one matches

## Before You Open a Pull Request

- check links between Markdown pages
- keep tables readable on GitHub
- make sure new examples are copyable and clearly labelled
- update [docs/glossary.md](docs/glossary.md) if you introduce important terms

## Scope Notes

This project is a learning guide, not a full engine reference. When in doubt, prefer clarity over completeness.
